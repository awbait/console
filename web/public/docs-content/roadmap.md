# Дорожная карта

Здесь собрано, что уже появилось на портале, что планируется и какие идеи рассматриваются. Это ориентир, а не обязательство: состав и приоритеты могут меняться, конкретные сроки не фиксируются.

> [!NOTE]
> Часть разделов портала (ИБ, сетевые политики, карта сети) сейчас работает в демонстрационном режиме и наполнится реальными данными вместе с сервисом policies. Ниже это отмечено.

## Уже появилось

- **Несколько версий сервиса** - у каждой версии чарта своя форма заказа и своё согласование; владелец выбирает, какие версии доступны для заказа и какая рекомендуется по умолчанию (см. [Публикация сервиса](publishing)).
- **Ускоренное обновление статусов** - помимо периодического опроса портал реагирует на события GitLab и Harbor через вебхуки, поэтому статус заказа меняется почти сразу (см. [Частые вопросы](faq)).
- **Панель администратора** - согласование публикаций, управление категориями каталога и экран состояния платформы (см. [Панель администратора](admin)).
- **Синхронизация с Git** - детект дрейфа, импорт сервисов, заведённых вне портала, и подтягивание состояния из Git (см. [Статусы и развёртывание](statuses)).

## Планируется

- **Сетевые политики (сервис policies)** - заказ правил доступа и трафика между сервисами. Сейчас раздел - заглушка (см. [Policies](policy)).
- **Раздел информационной безопасности** - согласование политик командой ИБ. Экраны уже есть, но работают на демонстрационных данных (см. [Информационная безопасность](security)).
- **Kyverno UI** - отчёты по политикам кластера: какие правила действуют и где зафиксированы нарушения, с разбивкой по namespace.
- **Карта сетевого взаимодействия** - визуальный редактор: при заказе политик будет видно, какой трафик между сервисами разрешён, а какой запрещён, ещё до применения в кластере. Пока это прототип на моковых данных (см. [Policies](policy)).
- **Интеграция с NetBox (IPAM)** - автоматическое выделение подсетей и адресов при заказе сервисов, которым они нужны. Пока не подключена (см. [Архитектура и интеграции](architecture)).
- **Управление найденными сервисами** - возможность взять под управление чарт, который портал нашёл в реестре автоматически: завести по нему публикацию и вывести в каталог.

## Идеи

Направления, которые пока на уровне обсуждения:

- **Day-2 операции** - рутинные операции с уже работающими сервисами (перезапуск, масштабирование) прямо из портала.
- **Резервное копирование (интеграция с Velero)** - UI над Velero для бэкапа ресурсов (PVC, объекты Kubernetes, виртуальные машины kube-virt) и баз данных с сохранением в S3.
- **Управление хранилищем (Ceph/S3)** - обзор всех проектов и их бакетов в Ceph и управление квотами на проект.
- **Мгновенная доставка изменений** - нативный вебхук GitLab в Argo CD, чтобы применение изменений не ждало периодического опроса.
- **Чат-ассистент (интеграция с LLM)** - встроенный помощник, который подскажет по каталогу, поможет заполнить форму заказа и разобраться в статусе.
- **Интеграция с EventMesh** - карта событий по аналогии с картой сетевого взаимодействия: как события ходят между сервисами через EventMesh.
