# Обзор портала

Console - это внутренний портал разработчика (Internal Developer Portal): UI-надстройка над GitOps, которая регламентирует и стандартизирует процесс заказа управляемых сервисов. Инструмент для внутреннего использования: команды через единую витрину сами заказывают сервисы, предоставляют их друг другу и следят за ними.

## Дополнительные функции для команд

Портал - это не только заказ сервисов. Поверх него встроены общие функции, которые платформа предоставляет командам и которые закрывают задачи, выходящие за рамки одной команды:

- **Поддержка.** Служба поддержки видит заказы всех команд, отслеживает проблемные и разошедшиеся с Git сервисы и помогает разбирать инциденты, не привязываясь к одной команде. См. [Панель поддержки](support).
- **Информационная безопасность.** Команда ИБ согласует сетевые политики и политики доступа и видит отчёты Kyverno о нарушениях в кластере - единое место контроля за правилами. См. [Информационная безопасность](security).

## GitOps-ядро

Под порталом - GitOps: ваш заказ превращается в изменение в Git (merge request), а кластер автоматически приводится к описанному состоянию. Git остаётся источником правды, а портал закрывает весь жизненный цикл сервиса - от заказа до вывода из эксплуатации.

## Возможности

- **[Каталог сервисов](catalog)** - просмотр доступных сервисов и их версий.
- **[Заказ сервиса](ordering)** - заказ нового экземпляра через форму и изменение параметров уже заказанного.
- **[Статусы и развёртывание](statuses)** - отслеживание развёртывания, история действий (аудит-след) и разбор расхождений с Git (дрейф).
- **[Публикация сервиса](publishing)** - вывод своих сервисов в каталог с формой заказа (для владельцев сервисов).
- **[Панель поддержки](support)** - сопровождение заказов всех команд (для поддержки).
- **[Панель администратора](admin)** - согласование публикаций и управление категориями каталога (для администраторов).
- **[Информационная безопасность](security)** - согласование политик и отчёты Kyverno (для ИБ).

## Роли и доступы

Доступ - производная от вашей организационной структуры, а не ручной список: роль и команды вычисляются из ваших групп в Keycloak при каждом входе. Вы работаете с заказами своих команд и не видите чужие. Подробнее - в разделе [Роли и доступы](roles).

> Если у вас нет ни одной `team-*` группы, портал покажет «нет группы» - обратитесь к администратору платформы за доступом.
